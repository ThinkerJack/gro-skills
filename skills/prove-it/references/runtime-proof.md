# Runtime Proof

用于需要证明“代码在真实运行环境中可用”的验证任务。具体 URL、账号、设备、
接口路径和环境变量属于项目上下文,不要写进通用 skill。

## 分层

1. Service smoke:访问非生产服务或本地服务,只输出状态码、响应类型和字段存在性。
2. Code-path probe:通过项目真实代码路径访问依赖服务,生成脱敏日志或报告。
3. Runtime probe:在应用进程、浏览器、模拟器、容器或设备上跑用户链路,读取项目已有
   结构化日志、trace、截图、录屏或报告复盘。

## 产物

- 原始证据:结构化日志、trace、测试报告、截图、录屏或监控查询结果。
- 摘要证据:状态、字段存在性、关键步骤结果和残余风险。

摘要只放状态、字段存在性、布尔证明和非敏感业务状态;不得放密钥、token、
cookie、个人数据或生产数据原文。

## 操作要点

- 先查项目文档、配置文件、测试脚本和 CI,找已有验证入口。
- 访问外部系统、真设备、付费资源或生产环境前必须确认显式开关或用户授权。
- 跳过真实探针后要立即结束该探针,避免继续执行副作用步骤。
- 能生成机器可读证据时优先生成;最终回复引用证据路径和关键结论。
