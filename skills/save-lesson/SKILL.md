# 记经验 save-lesson

## 核心原则

成长来自回流。重复出现的纠正，不应该继续靠记忆，而应该进入长期上下文或自动化检查。

## 什么时候用

- 一个任务结束后发现了可复用经验。
- 同一个错误、纠正或偏好出现第二次。
- 项目规则、命名、流程或验证方式被澄清。
- 失败暴露了 skill、pack、overlay 或工具链缺口。

## 不要用于

- 一次性细节，没有复用价值。
- 敏感信息、临时 token、私人数据。
- 还没验证的猜测；先用 `find-proof`。

## 输入

- 发生了什么。
- 触发原因和影响。
- 是否可复用。
- 应该写入的位置候选。

## 操作

1. 用一句话写清这次学到了什么。
2. 判断它是通用流程、能力扩展、项目规则还是自动化检查。
3. 选择沉淀位置：
   - 高频项目规则：`AGENTS.md` 或 overlay。
   - 业务词、领域词、项目黑话：`CONTEXT.md` 或项目知识库。
   - 架构和长期取舍：ADR 或设计记录。
   - 机械约束：测试、lint、脚本或 CI。
   - 通用流程缺口：core skill。
   - 工具或技术能力：pack。
4. 写出建议改动，不把一次性故事写成长期规则。
5. 如果信息不确定，标为待验证而不是规则。

## 输出

- 经验摘要。
- 是否可复用。
- 推荐沉淀位置。
- 建议写入内容。
- 是否需要自动化。

## 停止条件

经验有明确去处，或被判断为不值得沉淀。

## 升级条件

- 属于通用流程缺口：更新 core skill。
- 属于工具/技术能力：创建或更新 pack。
- 属于项目规则：更新 overlay 或项目说明。
- 能机械检查：优先做脚本、测试或 lint。

## 成功证据

下一次遇到同类任务时，AI 能从沉淀内容中少问一次、少错一次、更快验证，或被自动化规则直接挡住。
